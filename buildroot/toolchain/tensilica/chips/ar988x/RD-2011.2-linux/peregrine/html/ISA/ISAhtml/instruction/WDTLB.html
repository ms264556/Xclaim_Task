WDTLB — Write Data TLB Entry

Instruction Word

x24 : x24_Inst_0
2 3 1 2 1 1 8 7 4 3 0
12 4 4 4
0 1 0 1 0 0 0 0 1 1 1 0 s t 0 0 0 0

Assembler Syntax

WDTLB at, as

C Syntax

extern void XT_WDTLB(unsigned art, unsigned ars);

Description

(please consult the Xtensa ® Instruction Set Architecture Reference Manual for any cross references and additional information)

WDTLB writes the contents of address register at to the Data TLB entry specified by the contents of address register as. The Data TLB way, w, is specified by as bits wih..0, where wih is ceil(log2(DataTLBWayCount))-1. The index of the entry within that way is dependent on the configuration of the way selected, as it is taken from the same bits of as that would be used for translating a data virtual address, viz. bits eih..eil where eil is the least significant bit of the VPN configured for the way, and eih is eil + log2(DataTLBWay[w].IndexCount)-1. See Section 4.6.3.2 in the Xtensa ® Instruction Set Architecture Reference Manual for more information, Section 4.6.4.2 in the Xtensa ® Instruction Set Architecture Reference Manual for more information, and Section 4.6.5.5 in the Xtensa ® Instruction Set Architecture Reference Manual for more information for more information on the address and result register formats for specific Memory Protection and Translation options.

The point at which the Data TLB write is effected is implementation-specific. Any translation that would be affected by this write before the execution of an DSYNC instruction is therefore undefined.

WDTLB is a privileged instruction.

Operation

if CRING ≠ 0 then
	Exception (PrivilegedInstructionCause)
else
	(vpn, ei, wi) ← SplitDataTLBEntrySpec(AR[s])
	(ppn, sr, ring, ca) ← SplitDataEntry(wi, AR[t])
	DataTLB[wi][ei].ASID ← ASID(ring)
	DataTLB[wi][ei].VPN ← vpn
	DataTLB[wi][ei].PPN ← ppn
	DataTLB[wi][ei].SR ← sr
	DataTLB[wi][ei].CA ← ca
endif
